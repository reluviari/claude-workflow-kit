# /plan

Use this command to plan a task before implementation.

## Input expected

- goal
- scope
- constraints
- expected outcome

## Steps

1. Restate the objective.
2. Inspect relevant files.
3. Identify existing patterns.
4. Identify risks.
5. Propose a step-by-step plan.
6. List files to create or change.
7. Define tests/checks.
8. Stop before implementation.

## Output

Return:

- objective
- current understanding
- proposed plan
- files affected
- risks
- tests/checks
- confirmation needed before coding
