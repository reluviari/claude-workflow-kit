# /update-context

Use this command when project knowledge has changed.

## Steps

1. Identify stable knowledge learned during the task.
2. Decide where it belongs:
   - CLAUDE.md
   - docs/PROJECT_CONTEXT.md
   - docs/ARCHITECTURE.md
   - docs/DECISIONS.md
   - docs/TESTING_STRATEGY.md
3. Update only durable information.
4. Avoid temporary notes.

## Output

Return:

- files updated
- knowledge added
- why it matters
