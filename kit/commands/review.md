# /review

Use this command to review current changes.

## Steps

1. Inspect git diff.
2. Compare changes with the original objective.
3. Check correctness.
4. Check architecture.
5. Check tests.
6. Check security and edge cases.
7. Suggest minimal improvements.

## Output

Return:

- summary
- blocking issues
- non-blocking issues
- missing tests
- risk level
- approval recommendation
