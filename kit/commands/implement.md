# /implement

Use this command only after a plan exists.

## Steps

1. Read the approved plan.
2. Re-check relevant files.
3. Implement in small steps.
4. Preserve existing patterns.
5. Avoid unrelated refactors.
6. Run available checks.
7. Summarize changes.

## Stop conditions

Stop if:

- the implementation requires architectural change;
- the plan is incomplete;
- tests cannot be identified;
- public API contracts would change unexpectedly;
- database schema changes are needed but not approved.
