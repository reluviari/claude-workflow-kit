# Review Checklist

## Correctness

- Does the change solve the requested problem?
- Are edge cases handled?
- Is error handling adequate?

## Architecture

- Does it follow existing patterns?
- Is complexity justified?
- Are boundaries respected?
- Are SOLID principles reasonably preserved?
- Are dependencies pointing in the right direction?
- Are domain/application concerns separated from infrastructure concerns when applicable?

## Quality

- Is the code readable?
- Is naming clear?
- Is duplication avoided?

## Tests

- Are tests present when behavior changed?
- Are important scenarios covered?

## Security

- Are auth, permissions, validation and secrets handled correctly?

## Delivery

- Are docs updated when needed?
- Is the diff scoped?
