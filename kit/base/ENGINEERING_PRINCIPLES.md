# Engineering Principles

- Understand before changing.
- Plan before coding.
- Prefer small changes.
- Preserve existing conventions.
- Do not add dependencies without justification.
- Keep code simple and explicit.
- Use SOLID principles as a quality guide, not as ceremony.
- Prefer clear architectural boundaries and separation of concerns.
- Keep domain and business rules isolated from infrastructure when the stack supports it.
- Write tests for behavior changes.
- Review security and edge cases.
- Document decisions that affect architecture.
- Avoid unrelated refactors.
