# Refactor Workflow

Use this workflow when behavior must remain unchanged.

## Steps

1. Define refactor objective.
2. Confirm behavior remains unchanged.
3. Inspect affected files.
4. Identify safety checks.
5. Propose small steps.
6. Implement incrementally.
7. Run tests/checks.
8. Review diff for accidental behavior changes.

## Output

- refactor objective
- files changed
- behavior preservation notes
- tests/checks
- risks
