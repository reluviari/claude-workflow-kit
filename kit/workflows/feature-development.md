# Feature Development Workflow

Use this workflow for new features.

## Steps

1. Understand the business objective.
2. Extract requirements.
3. Identify acceptance criteria.
4. Inspect existing implementation.
5. Identify affected modules.
6. Propose technical plan.
7. Wait for approval.
8. Implement in small increments.
9. Add or update tests.
10. Run checks.
11. Review diff.
12. Update docs when needed.
13. Summarize delivery.

## Output

- objective
- requirements
- plan
- files changed
- tests
- risks
- next step
