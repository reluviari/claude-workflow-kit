# Release Review Workflow

Use this workflow before delivery or merge.

## Steps

1. Inspect git diff.
2. Review scope against objective.
3. Check tests and validation.
4. Check docs impact.
5. Check config/deployment impact.
6. Identify rollback concerns.
7. Produce final delivery summary.

## Output

- release summary
- included changes
- tests/checks
- risks
- rollback notes
- approval recommendation
