# Technical Design Workflow

Use this workflow before large implementation work.

## Steps

1. Define problem.
2. Define constraints.
3. Analyze current architecture.
4. Propose solution options.
5. Compare trade-offs.
6. Recommend one path.
7. Define implementation phases.
8. Define risks.
9. Define validation strategy.
10. Create or update docs/DECISIONS.md.

## Output

- problem
- context
- options
- decision
- trade-offs
- rollout plan
- risks
